## Выводы по сравнительному анализу

### 1. Яндекс.Геокодер — выбор для максимальной точности
- ✅ Эталонная точность координат (привязка к ФИАС + спутниковые данные)
- ✅ Полноценное обратное геокодирование
- ✅ Стабильная структура ответа
- ⚠️ Медленнее аналогов (~400 мс)
- ⚠️ Лимит 250 запросов/сутки на бесплатном тарифе

**Рекомендация**: Использовать как основной сервис для:
- Навигационных приложений
- Отображения объектов на карте
- Задач, где критична точность < 50 м

### 2. DaData — выбор для массовой обработки адресов
- ✅ Точность координат на уровне Яндекса (разница < 2 м)
- ✅ В 3-4 раза быстрее (~100 мс)
- ✅ Продвинутая нормализация и исправление опечаток
- ✅ Лимит 100 запросов/сутки на бесплатном тарифе
- ⚠️ Обратное геокодирование через отдельный API (Suggestions)

**Рекомендация**: Использовать как основной сервис для:
- Очистки пользовательского ввода в формах
- Пакетной обработки баз адресов (CRM, базы клиентов)
- Внутренних аналитических задач

### 3. GigaChat — не подходит для геокодирования
- ❌ Систематические ошибки координат (200 м — 8 км)
- ❌ Неструктурированный вывод (требует пост-парсинга)
- ❌ Риск «галлюцинаций» и устаревших данных
- ✅ Может использоваться для черновой разметки неструктурированных текстов

**Рекомендация**: Исключить из контура геокодирования.
